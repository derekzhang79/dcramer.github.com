Turn MySQL Completely Into UTF-8

Feb 21st, 2008

One of the developers at Curse was setting up a new database server today, and was trying to set all the defaults to UTF-8 for table creation/etc. While I knew about setting those, I always knew you still had to send the set names argument on your connection initialization. Well, it turns out, you don’t:


collation_server=utf8_unicode_ci
character_set_server=utf8
default-character-set=utf8
init-connect='SET NAMES utf8'

Slow Connections in Firefox 3 Beta 3 on Mac OS X Leopard

Feb 13th, 2008

Installed Firefox 3 Beta 3 here today at the office. “Shiny new fixes” they said. So I load it up, and obviously none of my addons work again (so I have to run them through my converter), and I attempt to hit Curse. “Connecting to www.curse.com…” for a minute. Ok, wtf, is the website down? Nope, Google doesn’t work either. Turns out, no websites loading in a timely fashion due to DNS lookups being extremely slow.

So I quickly Googled and found an answer (although unrelated it seemed).

Go to about:config and search for network.dns.disableIPv6. Change the value to true and voila, the internet is now fast again.

WRT54G and PS3 Internet Crashing

Feb 9th, 2008

As many people have recently, we experienced our internet constantly crashing while using the PS3. It turned out there were quite a few “solutions” via Google. At first we tried DMZing the router. We tried setting it between wired/wireless. Tried removing security and just locking to MAC addresses. Etc..

Recently, I found another solution posted online. Disable UPnP on your router. Voila no more crashing! It isn’t a solution for everyone, as from what I’m told, it will stop certain things from functioning like your media sharing via the PS3 and XBox 360. For us though, it’s a great solution, as we don’t use those and it will no longer crash the router each time we stop playing a game on the PS3.

If anyone has found any other solutions, let me know, but so far this seems to be working great :)
